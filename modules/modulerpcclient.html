External module ModuleRpcClient

      A stream provider. The stream provider is called until a stream emits a 'complete' or 'canceled' event, or the maximum number of retries, as per the exponential backoff schedule, has been reached.

        • (): Stream<Message>

        • Returns Stream<Message>

streamAsPromise

  • streamAsPromise<Message>(stream: Stream<Message>): Promise<Message[]>
    • Defined in client/stream.ts:200

    "Promisifies" a stream.

    example
    const result = await streamAsPromise(stream);
// If stream emits 1, 2, 3, result is [1, 2, 3].
// If the stream is canceled or errored, the promise is rejected.

    Type parameters

    • Message

      The type of the messages transmitted by the stream.

    Parameters

    • stream: Stream<Message>

      The stream to promisify.

    Returns Promise<Message[]>

    A promise that resolves to all the transmitted messages when the stream completes, and is rejected when the stream errors or is canceled (in this case, the error is a ClientRpcError of type ModuleRpcCommon.RpcErrorType.canceled).

streamFromArray

  • streamFromArray<T>(array: T[], error?: Error): Stream<T>
    • Defined in client/stream.ts:98

    Returns a stream created from an array that emits all the messages in the array then either completes or errors if error is specified.

    This function is useful for testing.

    example
    const stream = streamFromArray([1, 2, 3, 4]);
stream.on('message', console.log(message)).on('complete', () => {
  console.log('COMPLETE');
}).start();

const streamWithError = streamFromArray([1, 2, 3], new Error('error'));
streamWithError.on('error', err => {
  console.error('ERROR:', err);
}).start();

    Type parameters

    • Optional error: Error
