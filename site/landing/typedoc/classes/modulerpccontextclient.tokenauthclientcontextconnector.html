TokenAuthClientContextConnector

Provide a client context connector that adds an 'authorization' header.

The bearer token to put can be given and refreshed through the authenticate method (when initialized, the TokenAuthClientContextConnector operates without any token and so no 'authorization' header is added). Additionally, it is possible to specify an optional refreshTokenHandler that is automatically called expiryDateOffsetMs milliseconds before the current token expires.

    Constructs a token auth client context connector with a handler that will be called to refresh the token when it expires. The initial token must be provided with TokenAuthClientContextConnector.authenticate.

    Parameters

    • refreshTokenHandler: RefreshTokenHandler

      Given the refresh token of the current token info, returns a new token info in order to extend the life time of the authentication.

    • Optional expiryDateOffsetMs: undefined | number

      The refresh token handler will be called expiryDateOffsetMs milliseconds before the token expires.

    • Optional reportRefreshError: undefined | function

      Reports an error that occurred during the refresh (with auto refresh on, the refreshing occurred within a setTimeout and so in a different execution context).
