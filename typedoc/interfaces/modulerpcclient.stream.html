Stream<Message>

Streams are EventEmitter objects that are "read-only" (i.e. they can only register listeners, their emit function is not exposed), can be started and canceled, and expose a number of standard events relevant to the lifecycle of an RPC.

They are produced by a StreamProducer which takes an RPC method name and an RPC request.

Streams are used to implement both server streams and unary calls (a unary call is then simply a stream that is expected to send one and only one message).
